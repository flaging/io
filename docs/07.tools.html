Makefile 中:= ?= += =的区别

  • = 是最基本的赋值, 展开后文件的最终值
  • := 是覆盖之前的值, 运行到当前位置的值
  • ?= 是如果没有被赋值过就赋予等号后面的值
  • += 是添加等号后面的值

动态库依赖动态库，静态库依赖静态库，顺序

  • 编译链接库顺序:可执行文件->高级库->底层库,越底层越往后写;
  • 把第三方静态库链接给动态库需要加-fPIC
  • 链接过程:链接器从左往右搜索,未定义标注undefined,如果搜索到undefined的实现,就将其抽取出来,其余内容丢弃(依赖顺序的原因)
  • 环形链接:可以-la -lb -la这种方式解开
  • ld的path可以用-rpath指定
